# HTML mockups for Figma

Папка содержит статические макеты экранов системы в HTML/CSS для импорта в Figma.

## Состав
- `index.html` — оглавление всех экранов
- `styles.css` — общий UI-kit и базовые стили
- `login.html` — экран входа
- `employees.html` — список сотрудников + модалка создания
- `employee-card.html` — карточка сотрудника + вкладка пользователя
- `dictionaries.html` — страница справочников + модалка записи
- `process-registry.html` — реестр процессов
- `process-card.html` — карточка процесса + аналитика
- `process-version.html` — версия процесса, граф и модалка этапа
- `dashboards.html` — дашборды
- `profile.html` — профиль пользователя

## Импорт в Figma

### Вариант 1. HTML to Design
1. Установите в Figma плагин `HTML to Design`.
2. Откройте нужный HTML-файл в браузере.
3. Скопируйте его URL (`file:///...` или `http://127.0.0.1:5500/...`).
4. Запустите плагин и вставьте URL.
5. Импортируйте каждый экран отдельно, чтобы получить отдельные frame.

### Вариант 2. HTML to Figma
1. Установите любой плагин импорта HTML в Figma.
2. Если плагин не поддерживает `file:///`, откройте папку `figma_mockups` через локальный static server.
3. Используйте URL каждой страницы по отдельности.

## Практический порядок
1. Сначала импортируйте `index.html`.
2. Затем по отдельности: `login.html`, `employees.html`, `employee-card.html`, `dictionaries.html`, `process-registry.html`, `process-card.html`, `process-version.html`, `dashboards.html`, `profile.html`.
3. После импорта выделите повторяющиеся элементы и превратите их в компоненты Figma: sidebar, кнопки, поля ввода, карточки, таблицы, модалки.

## Рекомендации
- Один HTML-файл = один основной экран.
- Для диплома удобнее собирать на их основе отдельные high-fidelity frames.
- При необходимости можно доработать любой макет и импортировать его повторно.
